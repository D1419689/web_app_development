# 產品需求文件 (PRD)：任務管理系統

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常有許多待辦事項需要處理。然而缺乏一個簡單且美觀的系統來追蹤任務，容易導致遺漏重要事項或工作效率低落。
- **目標用戶**：個人用戶（如學生、上班族、自由工作者）。
- **核心價值主張**：提供一個輕量、直覺且具備現代美感（動態與流暢操作）的工具，讓使用者可以快速記錄、追蹤並管理日常任務，有效提升個人生產力並降低任務焦慮。

## 2. 功能需求

以下為預計開發的主要功能清單與使用者故事：

1. **基本任務管理（新增、完成、刪除與篩選）**
   - 作為個人用戶，我希望可以新增待辦列表、標記任務為完成、刪除不需要的任務，以及篩選視圖（全部、未完成、已完成），以便有效管理我的待辦事項。
2. **任務期限設定**
   - 作為個人用戶，我希望能夠為任務設定截止日期，並且在介面上看到即將到期或已過期的視覺提示（如紅色警示），以便我安排處理的優先順序。
3. **任務分類或標籤**
   - 作為個人用戶，我希望能建立不同的分類（如工作、生活、學習）或添加自訂標籤，以便將相關的任務分組管理，畫面不雜亂。
4. **任務搜尋**
   - 作為個人用戶，我希望可以使用關鍵字搜尋任務標題，以便在累積大量任務時能快速找到特定的待辦事項。
5. **數據總覽與成就統計**
   - 作為個人用戶，我希望可以看見簡單的進度統計（例如：今日已完成任務數、進度條），以便獲得成就感並了解自己的執行狀況。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 作為後端與靜態模板引擎，資料庫使用關聯式資料庫 SQLite。前端應避免龐大框架，使用 Vanilla CSS 與原生 JavaScript 處理動態交互。
- **效能考量**：操作需輕量且反應快速。
- **介面設計**：需符合現代高品質網頁設計（Modern Web Design），包含動態懸停效果（Hover effects）、和諧的色彩計畫，或許考慮玻璃擬物化（Glassmorphism）與滑順的微動畫（Micro-animations）。
- **安全考量**：防範基本的 Web 攻擊（如 XSS, CSRF）。

## 4. MVP 範圍 (最小可行性產品)

- **Must Have (必須有)**
  - 任務的 CRUD（新增、讀取/列表、刪除）。
  - 任務狀態標記（已完成/未完成）與狀態篩選。
  - SQLite 基礎資料表的建置。
- **Should Have (應該有)**
  - 任務截止日期與基本的逾期警示樣式。
  - 高質感的 UI 與現代化字體（如 Google Fonts - Inter 或 Roboto）。
- **Nice to Have (可以有)**
  - 全文關鍵字搜尋。
  - 資料總覽進度條。
  - Dark Mode（深色模式）支援。
  - 任務群組/分類標籤。
